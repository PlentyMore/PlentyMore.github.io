Java Queue接口常用实现类

Queue表示队列，在Java中，一般是FIFO（先进先出）的队列，但也有不是先进先出的队列，比如优先级队列

接口

队列也可以分为阻塞队列和非阻塞队列，则阻塞队列新增了几个方法

  • 不阻塞 调用之后如果操作失败（一般都是因为队列容量不足或者队列为空），则立即返回，不发生阻塞，add(e)，offer(e)remove(e)，poll()，element()，peek()等，在Queue接口的方法一般都是不阻塞的

  • 阻塞 调用之后如果暂时无法操作成功，则一直阻塞到操作成功，put(e)，take()等

  • 限时阻塞 调用之后如果暂时无法操作成功，则阻塞一段时间，超时后仍为操作仍然未成功则返回，offer(e,timeout,unit)，poll(timeout,unit)等

非线程安全的实现类

PriorityQueue

PriorityQueue是优先级队列，它不是FIFO的，而是根据元素的优先级，优先级高的或者低的先出队列，它是非阻塞队列。不允许存放null元素

  • 构造器

  • 默认值
    默认的队列容量为11

非线程安全的Queue实现类就只有这一个（不包括Deque的实现类，Deque拓展了Queue接口），其他的大部分都是一些内部类

线程安全的实现类

PriorityBlockingQueue

PriorityBlockingQueue是无边界的阻塞队列，它和PriorityQueue一样能根据Comparator按照特定的顺序将元素出队列。不允许存放null元素

  • 构造器

  • 默认值
    默认容量为11（因为底层是数组，而这是一个无边界队列，所以随着元素的增加需要进行扩容操作）

  • 底层存储结构
    底层为数组，扩容策略和PriorityQueue是一样的

ArrayBlockingQueue

ArrayBlockingQueue是有边界（即有容量限制）的阻塞队列，基于数组实现。不允许存放null元素

  • 构造器

  • 默认值
    无，队列容量的值必须在构造器指定

  • 底层存储结构
    底层是数组，容量指定之后就无法改变

LinkedBlockingQueue

LinkedBlockingQueue是一个阻塞队列，它既可以是有边界的，也可以是无边界的。不允许存放null元素

  • 构造器

  • 默认值
    队列容量capacity默认为Integer.MAX_VALUE，也就是默认为无边界队列

  • 底层存储结构
    底层存储结构为链表

ConcurrentLinkedQueue

ConcurrentLinkedQueue是无边界的阻塞队列，基于链表实现，使用了无锁算法提高并发性能。不允许存放null元素

  • 构造器

  • 默认值
    无

  • 底层存储结构
    底层存储结构为链表

SynchronousQueue

SynchronousQueue是一个阻塞队列，它的特点是在插入元素的时候必须有另一个线程在等待取出元素，或者反过来，它没有容量的概念，因为这个队列是不存储元素的，每次将一个元素入队列的时候必须在另一个线程将其取出才能把下一个元素入队列。不允许存放null元素

  • 构造器

  • 默认值
    无

  • 底层存储结构
    底层存储结构为Transferer

LinkedTransferQueue

LinkedTransferQueue是无边界的传输队列（TransferQueue。不允许存放null元素
